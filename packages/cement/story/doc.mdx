# Foundation CSS

The Konstruct Foundation is foundation layer css library that sets a good browser reset, and then adds minimal styling on base html elements to give you a good head start in your application.

It's built to be minimally opinionated with most visual decisions stored as css variables that can easily be overridden.

- [Core](#core)
- [Parts](#parts)
- [Theming](#theming)

<Source

  `}

### Coverage

This library is meant to cover the base html foundation, but that's an ongoing process still. Currently this library provides styling for:

* CSS Variables for common things
* Browser reset
* Typography foundation
* Form input elements

**Note:** This library doesn't want to make any opinions that can affect layout as far as possible, so we don't expect to add anything like grids or specific spacing on containers (`div, section, header, form, etc`);

## Parts

Parts are small reusable bits of functionality, usually used to build up complete features. 

- [Buttons](#buttons)

### Buttons

#### Installation

**Installing the library:** ```yarn add @traaidmark/konstruct-parts-css;```

* **With js:** ```import '@traaidmark/konstruct-parts-css/buttons';```
* **With HTML:** ```<link rel="stylesheet" type="text/css" href="" />```
* **With css:** ```@import ...```

#### Usage

The ```.part-button``` can be applied equally to ```<a>``` & ```<button>``` elements, and of course will work with most JS link libraries too.

<Source
  code={dedent`
    <a href="#" className="part-button">Button</a>
    <button className="part-button">Button</button>
  `}
/>

#### Variables

This library provides a set of variables to help you work consistently with buttons. You can override these at root level or use them to style your own buttons.

<Source
  code={dedent`
    --button-radius: var(--radius-small);
    --button-border-width: 2px;
    --button-text-color: var(--color-blue);
    --button-text-color-hover: var(--color-darker);
    --button-bg-color: var(--color-darker);
    --button-bg-color-hover: var(--color-blue);
    --button-border-color: var(--color-darker);
    --button-border-color-hover: var(--color-blue);
  `}
/>

#### Theming

Since buttons are styled with variables, you can easily create your own button styles like this example:

<Source
  code={dedent`
    .part-button-danger {
      --button-text-color: var(--color-darker);
      --button-text-color-hover: var(--color-red);
      --button-bg-color: var(--color-red);
      --button-bg-color-hover: var(--color-darker);
      --button-border-color: var(--color-red);
      --button-border-color-hover: var(--color-darker);
    }
  `}
/>

## Theming

This library strives to follow the [BEM](https://css-tricks.com/bem-101/) methodology to manage & maintain our css codebase.

Although, of course you can do whatever you want, I would like to highly recommend that you do the same when you create your own classes on top of mine.

Most elements within this library is built on top of css variables, which means extending an element is usually as simple as creating a new modifier that overrides some variables:

<Source
  code={dedent`
    .part-flag--customFlag {
      --flag-bg-color: var(--color-blue);
      --flag-text-color: var(--color-darker);
    }
  `}
/>
